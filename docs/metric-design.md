# 指标设计说明

## 1. 设计原则

- 只对“可被当前数据直接支撑”的维度打分
- 每个维度必须能追溯到明确特征和公式
- 对数据不足维度不强算，直接跳过并说明原因
- 总分只基于已支持维度归一化聚合，避免缺失惩罚

## 2. 维度选择结果

### 已实现维度

1. 品牌成熟度
2. 产品质量
3. 市场需求匹配度
4. 创新力
5. 运营韧性
6. 性价比

### 跳过维度

1. 市场趋势契合度
2. 可持续发展
3. 供应链可靠性

跳过原因：缺乏趋势词库、ESG 认证、库存履约等直接数据。

## 3. 维度定义依据

| 维度 | 业务目标对应 | 核心输入 |
|---|---|---|
| 品牌成熟度 | 判断品牌稳定性与市场存在感 | 品牌成立时间、访问量、社媒与站点功能覆盖 |
| 产品质量 | 判断产品口碑与感知质量 | 评论均分、高分占比、甜度/人工甜味投诉 |
| 市场需求匹配度 | 判断市场是否认可品牌供给 | 月访问量、品牌词搜索量、销售估算 |
| 创新力 | 判断品牌是否保持产品与增长动能 | 新品信号、融资轮次、增长预测 |
| 运营韧性 | 判断品牌运营触达和渠道稳定性 | 物流承运商、销售渠道、联系方式、地域集中度 |
| 性价比 | 判断用户是否认可价格与价值 | 评论均分、价格抱怨、价格样本 |

## 4. 计算逻辑

### 4.1 品牌成熟度

公式：

```text
品牌成熟度 =
0.35 * 年限得分 +
0.40 * 流量得分 +
0.25 * 覆盖度得分
```

其中：

- 年限得分 = `min(品牌存在年限 / 10 * 100, 100)`
- 流量得分 = `min(estimated_visits / 2,000,000 * 100, 100)`
- 覆盖度得分 = 社媒数、站点功能数、联系方式数的组合得分

业务意义：识别品牌是否已从“新品牌”进入“稳定经营品牌”阶段。

### 4.2 产品质量

公式：

```text
产品质量 =
0.55 * 星级得分 +
0.30 * 高分占比得分 +
0.15 * (100 - 投诉惩罚)
```

其中：

- 星级得分 = `评论均分 / 5 * 100`
- 高分占比得分 = `4-5 星评论占比 * 100`
- 投诉惩罚 = 甜度抱怨率与人工甜味抱怨率加权

业务意义：反映用户对产品真实口味和体验的感知结果。

### 4.3 市场需求匹配度

公式：

```text
市场需求匹配度 =
0.35 * 流量得分 +
0.25 * 搜索得分 +
0.40 * 销售代理得分
```

其中：

- 流量得分 = `最近月访问量 / 500,000 * 100`
- 搜索得分 = `品牌词搜索量 / 100,000 * 100`
- 销售代理得分 = `estimated_sales_yearly / 8,000,000,000 * 100`

业务意义：判断品牌供给与消费者注意力是否匹配。

### 4.4 创新力

公式：

```text
创新力 =
0.35 * 新品信号得分 +
0.30 * 融资得分 +
0.35 * 增长动能得分
```

其中：

- 新品信号得分 = 最近一年商品发布与公司产品信号
- 融资得分 = `融资轮次 / 8 * 100`
- 增长动能得分 = funding prediction 概率映射

业务意义：反映品牌是否持续推出新内容并获得资本与市场认可。

### 4.5 运营韧性

公式：

```text
运营韧性 =
0.75 * 渠道能力得分 +
0.25 * (100 - 集中度惩罚)
```

其中：

- 渠道能力得分 = 物流承运商、销售渠道、联系方式的组合得分
- 集中度惩罚 = Top 国家流量占比映射的风险扣分

业务意义：反映品牌在触达、服务与市场分布上的抗波动能力。

### 4.6 性价比

公式：

```text
性价比 =
0.45 * 评价得分 +
0.35 * 价格情绪得分 +
0.20 * 价格水平得分
```

其中：

- 评价得分 = `评论均分 / 5 * 100`
- 价格情绪得分 = `1 - 价格抱怨率`
- 价格水平得分：依据样本均价是否落入合理区间给定经验分

业务意义：判断消费者是否认为“这个价格值得”。

## 5. 权重设计

| 维度 | 权重 |
|---|---:|
| 品牌成熟度 | 0.22 |
| 产品质量 | 0.22 |
| 市场需求匹配度 | 0.20 |
| 创新力 | 0.13 |
| 运营韧性 | 0.13 |
| 性价比 | 0.10 |

理由：

- 品牌成熟度、产品质量、需求匹配度最接近品牌健康度主问题，因此权重最高
- 创新力和运营韧性重要但受数据纯度影响稍大，因此采用中等权重
- 性价比受 merch 污染影响最大，因此保留但降低权重

## 6. 置信度设计

### 计算思路

- 高置信度：原始样本较直接、结构清晰、跨源能相互验证
- 中置信度：可计算，但存在口径差异或样本不足
- 低置信度：样本受污染或业务指向不完全稳定

### 结果解释

- `>= 0.80`：可作为核心判断
- `0.60 - 0.79`：可作为辅助判断
- `< 0.60`：仅作参考，需补数验证

## 7. 指标优化考量

### 性能优化

- 所有计算基于标准库，启动和运行开销低
- 通过统一 `FeatureBundle` 避免多次重复解析 JSON
- 输出格式固定，便于后续接入 API 或前端展示

### 业务适应性优化

- 维度函数可单独替换，便于迁移到其他品牌
- 价格、流量、搜索量阈值集中在评分函数内，便于后续参数化
- LLM 节点允许真实 API 与本地 fallback 双模式运行

## 8. 已知限制

- 商品样本不能完全代表饮料主 SKU
- 评论量偏少，不能推导严谨的人群差异
- 没有真实订单和退货数据，无法直接建模复购与履约质量
- 可持续发展维度缺少可核验字段，因此未纳入总分
