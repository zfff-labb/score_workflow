# 数据探索与质量评估

## 1. 数据来源分析

| 文件 | 数据主题 | 来源特征判断 | 可靠性评估 | 使用边界 |
|---|---|---|---|---|
| `brand-info.json` | 官网、电商站点、社媒与运营特征 | 类似品牌监测/电商情报聚合结果 | 中高 | 适合做品牌成熟度、渠道覆盖和站点运营画像，不适合替代真实财务审计 |
| `company-info.json` | 公司融资、投资人、时间线、技术与增长信号 | 类似公司情报/投研平台导出 | 中高 | 适合做成长阶段、融资活跃度与创新信号分析，不适合精确财务建模 |
| `products-info.json` | Shopify 商品快照 | 电商商品接口或站点抓取 | 中 | 适合做商品结构和价格快照，但本样本中 merch 占比高，不能直接代表核心饮料货架 |
| `review-info.json` | 文本评论与星级 | 评论系统或用户调研样本 | 中 | 适合做轻量口碑分析，样本量偏小，不适合做统计显著性结论 |
| `traffic-info.json` | 流量、来源、关键词、国家分布 | 流量估算平台输出 | 中 | 适合做趋势和结构判断，不应视为一手埋点数据 |

## 2. 数据质量评估

### 2.1 完整性检查

- `brand-info.json`：关键字段齐全，包含品牌名称、创建时间、站点特征、社媒、流量与销售估算。
- `company-info.json`：融资轮次、投资人数量、公司类型、时间线、产品概览均存在。
- `products-info.json`：包含 `products` 顶层数组，商品、变体、图片、选项字段可用。
- `review-info.json`：50 条评论均含文本和星级，时间字段完整。
- `traffic-info.json`：流量来源、国家分布、关键词和月访问量均可用。

### 2.2 准确性与一致性检查

- 品牌名称在多个文件中一致指向 `OLIPOP` / `drinkolipop.com`。
- 评论平均分与 `brand-info` 中的 Trustpilot 平均分不一致，说明评论来源不同，不能混为同一口碑池。
- `products-info` 中“商品”与“饮料 SKU”不完全等价，存在大量节日周边，属于语义层面的一致性风险。
- `traffic-info` 与 `company-info` 的月访问数据口径不同，一个更像 Similarweb 快照，一个更像 Semrush 指标，应作为不同来源信号联合使用。

### 2.3 主要质量风险

1. 商品样本偏移：`products-info` 以 merch 为主，会影响价格、创新和产品广度判断。
2. 评论样本有限：50 条评论足够做主题归纳，但不适合做细粒度人群细分。
3. 流量为估算值：可反映量级和渠道结构，但不用于精确 KPI 考核。
4. ESG 与供应链数据缺失：无法支持可持续发展和供应链可靠性维度的强结论。

## 3. 数据预处理步骤

### 3.1 清洗

- 统一日期格式，将 ISO 字符串解析为 `datetime`
- 将价格、访问量、搜索量等字段转换为数值类型
- 将评论文本统一转为小写，去除标点，便于关键词统计
- 统一 `tags` 字段格式，兼容字符串和数组

### 3.2 转换

- 从原始评论中提取：
  - 平均星级
  - 高分占比和低分占比
  - 价格抱怨率
  - 甜度抱怨率
  - 人工甜味/假甜感抱怨率
- 从商品数据中提取：
  - 商品数
  - 变体数
  - 价格最小值、最大值、均值
  - 饮料类与 merch 类粗分类数量
- 从流量数据中提取：
  - 最近月访问量
  - Direct/Search/Social 占比
  - Top 国家流量占比
  - 品牌词搜索量

### 3.3 集成

- 将品牌、公司、商品、评论和流量信息汇总为统一 `FeatureBundle`
- 在评分层只消费标准化特征，不直接依赖原始 JSON 结构
- 为每项风险增加数据质量说明，供最终报告展示

## 4. 异常值处理

### 4.1 检测方法

- 结构异常：检查关键顶层键是否存在
- 语义异常：识别 merch 商品对饮料目录的污染
- 数值异常：价格区间过宽、流量集中度过高
- 样本异常：评论数偏小、跨来源指标口径不一致

### 4.2 处理策略

- 不直接删除异常样本，而是通过“置信度下降”和“维度跳过”控制风险
- 对受污染严重的维度降低权重解释强度，例如性价比和创新力
- 对无法支撑的维度显式 `skipped`，避免构造伪指标

### 4.3 处理理由

- 面试任务更重视判断边界和可解释性，而非把所有维度强行打分
- 对数据缺口显式声明，比看似完整但不可靠的分数更符合业务使用场景

## 5. 探索性分析结果

### 5.1 数据规模

- 评论样本数：50
- 商品样本数：30
- 商品变体数：190
- 融资轮次：7
- 投资人数量：46

### 5.2 评论洞察

- 评论均分约 `4.18/5`
- 5 星评论占比高，整体口碑偏正向
- 高频主题集中在 `taste`、`sweet`、`apple`、`cola`、`health`
- 主要负面点集中在：
  - 太甜
  - 人工甜味/后味
  - 价格偏高

### 5.3 商品洞察

- 商品快照中 merch 占显著多数，核心饮料商品只有少量显性条目
- 价格范围较宽，说明样本里混入非标准饮料商品
- 不适合直接用全部商品均价代表饮料价格竞争力

### 5.4 流量洞察

- 最近月访问量约 `402,363`
- 流量来源以 `Direct` 和 `Search` 为主，品牌心智较强
- 美国流量占比约 `71%`，说明核心市场聚焦美国

### 5.5 商业成长洞察

- 公司处于较成熟的成长阶段，已有多轮融资和较多投资人
- 既有官网流量，也有投研平台记录的增长信号
- 适合评估为“高认知、高热度、但部分维度仍需补强数据”的品牌
